## Section notes

For each page section, provide:

- section purpose
- approved copy source
- preferred starter section
- variant or preset preference
- media assets
- CTA destination
- proof or credibility assets
- responsive notes

## Brand and theme notes

Provide:

- color direction
- typography direction
- imagery style
- surface/radius feel
- motion expectations
- examples to emulate
- examples to avoid

## Integration notes

List:

- contact forms
- booking links
- newsletter tools
- analytics
- pixels or third-party scripts
- CRM or form destinations
- required environment variables

## Agent start checklist

Before coding starts, confirm:

- approved inputs are present or waived
- placeholder policy is clear
- new dependencies are not required or are approved
- no CMS/auth/database assumption is hidden
- client-specific work will stay in the client repo
- reusable system changes are explicitly approved
